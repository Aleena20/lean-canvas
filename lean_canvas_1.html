August Home — Smart Access for Short-Term Rental Hosts

Smart Access Innovators · EU Business School Munich · Term III 2025–26 · Customer segment: multi-property Airbnb / Booking.com hosts in the EU
Slibrary references: 08 (Lean Startup Core) · 14 (Value Proposition) · 16 (Customer Segments & GTM)

Slibrary 08
1. Problem
Top 3 pains (to be validated)
Hosts spend 20–40 min per booking on manual guest access coordination — WhatsApp codes, lockbox PINs, physical key handovers
No existing smart lock (Ring, Schlage, August) auto-generates and revokes access codes in sync with Airbnb / Booking.com calendars
GDPR friction: platforms requiring guest account creation or biometric data to grant access — guests refuse, check-in fails
Existing alternatives
Physical lockboxes — insecure, no audit trail
Manual smart lock codes — host-managed, not automated
PMS tools (Guesty, Hospitable) — complex, expensive, not built for solo hosts
1
Slibrary 08
2. Solution
MVP features
Auto-generate time-limited digital key when booking confirmed on Airbnb / Booking.com — auto-revoke at checkout with zero manual steps
Single mobile dashboard to manage guests, cleaners, and maintenance across all properties with individual time windows
GDPR-compliant access audit log exportable for EU night-cap enforcement (Vienna 90d, Barcelona 30d, Berlin 3mo)

Slibrary 14
3. Unique value proposition
Single clear message
Your Airbnb calendar controls your locks — automatically, compliantly, without a locksmith.
High-level concept
The only smart access platform built for the EU regulatory environment — GDPR-native, night-cap audit-ready, retrofit-first.

Slibrary 14
4. Unfair advantage
Cannot be easily copied or bought
Existing Alexa, Google Home, and HomeKit integrations — platform relationships competitors must rebuild from zero
Matter-native hardware base — interoperates with the entire smart home stack without proprietary lock-in
EU compliance architecture: GDPR-native design and night-cap audit log are not on Ring or Schlage product roadmaps

Slibrary 16
5. Customer segments
Early adopter beachhead
Multi-property STR hosts — 2–5 listings on Airbnb / Booking.com in EU cities with active night-cap limits
Operationally overwhelmed, technically comfortable, managing cleaners and maintenance remotely
Mainstream target (post-chasm)
Casual STR hosts — 1 property, motivated by guest experience ratings and contactless check-in

8. Cost structure
Fixed costs
Product and engineering: booking-platform sync API, mobile dashboard, GDPR compliance architecture
Customer success and onboarding for early STR host cohort
Variable costs
Cloud infrastructure: access event logging, calendar sync API calls, audit log storage per property
Customer acquisition: STR community outreach, content marketing
Hardware COGS: per-unit manufacturing and logistics for retrofit lock

9. Key metrics
Innovation accounting
Activation: % of users completing first automated booking sync within 7 days of setup
Retention: monthly churn per property subscription — target <5%
Revenue: MRR per cohort; LTV:CAC ratio — target >3:1
Referral: new host sign-ups via existing host recommendation — low CAC channel
Validation: % of interviewed hosts confirming access coordination as top-3 pain point
